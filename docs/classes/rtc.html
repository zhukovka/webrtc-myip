setSourceVideo

  • setSourceVideo(type?: MediaType, videoElement: HTMLVideoElement, mediaConstraints?: MediaStreamConstraints): Promise<MediaStream>
    • Defined in RTC.ts:427

    Sets the Presenter's stream source from either webcam or display and related HTMLVideoElement. It returns a Promise that resolves to a MediaStream object. If the user denies permission, or matching media is not available, then the promise is rejected with NotAllowedError or NotFoundError respectively.

    Example:

    const mediaType = 'userMedia';
const userMedia = document.getElementById("userMedia");
rtc.setSourceVideo('userMedia', userMedia);

    Parameters

    • Default value type: MediaType = "userMedia"
    • videoElement: HTMLVideoElement
    • Optional mediaConstraints: MediaStreamConstraints

      optional parameter. Defaults to { audio: true, video: true } for user media (webcam) and is NOT configurable for screen sharing as audio is not supported https://blog.mozilla.org/webrtc/getdisplaymedia-now-available-in-adapter-js/

    Returns Promise<MediaStream>
